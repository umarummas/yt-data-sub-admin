## Quick Integration With Current Backend

Use this admin panel against the backend in this workspace (`yt-data-sub-backend`).

1. Backend base URL is expected to be `http://localhost:5000/api/v1`.
2. Admin routes are expected at `http://localhost:5000/api/v1/admin`.
3. Configure via env file in this project root:

```bash
# yt-data-sub-admin/.env
VITE_API_BASE_URL=http://localhost:5000/api/v1
VITE_USE_LOCAL_API=true
```

4. Start backend first:

```bash
cd ../yt-data-sub-backend
npm install
npm run dev
```

5. Start admin panel:

```bash
cd ../yt-data-sub-admin
npm install
npm run dev
```

6. Open admin app:

```text
http://localhost:5173
```

## 🗺️ Admin Routes Overview

### Backend Routes
All admin routes are prefixed with `/api/v1/admin` and require `authMiddleware` authentication.

**File**: `/backend/src/routes/admin.routes.ts`

```typescript
// Authentication
POST   /login                           // Admin login

// Dashboard
GET    /dashboard                       // Get dashboard statistics

// User Management
GET    /users                           // Get all users (paginated)
GET    /users/:id                       // Get user by ID
PUT    /users/:id                       // Update user details
PUT    /users/:id/status               // Update user status (active/inactive)
DELETE /users/:id                       // Delete user

// Wallet Management
POST   /wallet/credit                   // Manually credit user wallet

// Audit Logs
GET    /audit-logs                      // Get audit logs (paginated)
DELETE /audit-logs/:id                  // Delete audit log

// Pricing Management
GET    /pricing                         // Get all pricing plans (filtered)
GET    /pricing/:id                     // Get plan by ID
GET    /pricing/provider/:providerId    // Get plans by provider
POST   /pricing                         // Create new plan
PUT    /pricing/:id                     // Update plan
DELETE /pricing/:id                     // Delete plan
POST   /pricing/bulk-import            // Bulk import plans from array
```

## 🎨 Admin UI Pages

### 1. Login Page
- **Path**: `/login`
- **Features**:

  - Form validation
  - Error handling
  - Redirect to dashboard on success

### 2. Dashboard
- **Path**: `/dashboard`
- **Features**:
  - 📊 Total Users count
  - 👥 Active Users count
  - 💳 Total Transactions count
  - ✅ Successful Transactions count
  - Real-time data from backend

### 3. Users Management
- **Path**: `/users`
- **Features**:
  - 📋 User list (paginated)
  - 👁️ View user details
  - ✏️ Edit user information
  - 🔄 Update user status
  - 🗑️ Delete user

**User List Columns**:
- Name
- Email
- Phone
- Status (Active/Inactive)
- KYC Status
- Actions (View, Edit, Status, Delete)

### 4. Pricing Plans
- **Path**: `/pricing`
- **Features**:
  - 📋 All pricing plans with pagination
  - 🔍 Filter by Provider (MTN, Glo, Airtel, 9mobile)
  - 🔍 Filter by Type (AIRTIME, DATA)
  - ➕ Create new plan
  - ✏️ Edit existing plan
  - 🗑️ Delete plan
  - 📤 Bulk import from JSON/CSV
  - Status badge (Active/Inactive)

**Plan List Columns**:
- Plan Name
- Provider
- Type
- Price (₦)
- Discount (%)
- Status
- Actions (View, Edit, Delete)

**Create/Edit Plan Modal**:
- Provider selection
- Plan name
- Price input
- Discount percentage
- Plan code
- External plan ID
- Active/Inactive toggle
- Form validation

**Bulk Import Modal**:
- JSON format support
- CSV format support
- Sample file downloads
- Validation & error messages
- Import progress indication

### 5. Wallet Credit
- **Path**: `/wallet-credit`
- **Features**:
  - 👤 User selection dropdown
  - 💰 Amount input
  - 📝 Description/reason input
  - ✅ Success notifications
  - ❌ Error handling
  - Transaction summary before submission
  - User info preview

### 6. Audit Logs
- **Path**: `/audit-logs`
- **Features**:
  - 📋 All audit logs (paginated)
  - 📅 Timestamp display
  - 👤 Admin who performed action
  - 📝 Action description

### 7. Profile/Settings

## ❌ Error Handling

### Common Error Responses

#### 401 Unauthorized
```json
{
  "success": false,
  "message": "Invalid credentials",
  "statusCode": 401
}
```

#### 403 Forbidden
```json
{
  "success": false,
  "message": "Account is inactive",
  "statusCode": 403
}
```

#### 404 Not Found
```json
{
  "success": false,
  "message": "User not found",
  "statusCode": 404
}
```

#### 400 Bad Request
```json
{
  "success": false,
  "message": "Missing required fields: providerId, providerName, name, price, type",
  "statusCode": 400
}
```

#### 500 Server Error
```json
{
  "success": false,
  "message": "Internal server error message",
  "statusCode": 500
}
```
